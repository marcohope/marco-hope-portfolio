What Makes a Good Newsletter?

When designing a newsletter here are some key elements it should feature:

Clear Purpose: Know what you want the reader to do: buy, read, follow, etc

Strong Visual Hierarchy: Easy to scan with clear headings, bite-sized text that is readable, good use of spacing, etc.

Consistent Branding: Use Minoa’s brand fonts, colours, tone, and logo consistently. 

Engaging, Relevant Content: Only include what matters. Make sure the content under the headers is relevant to the body. If it’s fluff, cut it.

Clear CTA (Call to Action): Every newsletter should guide readers towards 1-2 simple actions. It can direct them towards one product, multiple products, to an informational page like about us. Don’t overdo CTA and give the readers a reason to engage.

Mobile-Friendly Design: Over 60% of emails are opened on phones. Make sure the layout is responsive especially if you’re importing a design from a third party (figma, photoshop, etc).

# FM-Radio-Module

GITHUB: https://github.com/marcohope/FM-Radio-Module

## Overview

This project demonstrates the creation of a functional FM radio using the TEA5756 FM module, Arduino Due, Rotary Encoder, OLED Display, Audio Amplifier, and a speaker. The system was designed to tune into radio frequencies, display station information, and output sound through the speaker.

## Features

FM Tuner: Allows users to tune into FM radio stations using a Rotary Encoder. OLED Display: Displays the current radio frequency in real-time. Audio Amplification: Outputs clear audio through an amplifier and speaker. User Interface: Simple and intuitive interface for tuning stations.

## Components

Arduino Due: The microcontroller used for handling inputs and managing communication between components. TEA5756 FM Module: Used for receiving FM radio signals. Rotary Encoder: Enables users to adjust the radio frequency. OLED Display: Shows the current frequency being tuned. Audio Amplifier: Amplifies the audio output from the FM module. Speaker: Outputs the audio.

## Software

The project was developed using C++ to handle hardware interaction and signal processing.

## Schematic & Workflow

I used Tinkercad and MS Paint to create the system's schematic. The project planning and functional breakdown were organized using a flowchart, outlining how the radio operates and interacts with the user.

## Auto Watering System With Arduino

Minor Project: Auto Plant Watering with Arduinio and Matlab

ABSTRACT
INTRODUCTION: Moving into 2023 as a whole, we should look towards more economically friendly alternatives in all things we do, and reducing water usage should be one of them. More specifically, people often use insufficient or excessive water when watering their plants —but what if this was made easier?
 OBJECTIVES: My objective for this projective is to create a program via Matlab that allows me to use my Arduino to monitor the moisture levels of the soil of the plant, which will tell me if it needs to be watered or not if the plant needs to be wet the moisture sensor will detect the dry level and send water into the soil until it reacts a specific moisture level.
DESIGN ANALYSIS: To create this watering system, I will need to use a 9v battery & cables, Grove Beginner Kit, a container for water, a water pump to deliver water from the water container to the plant, a MOSFET Grove board, a potted plant, a moisture sensor and a container for spills.
PROCEDURE: I used Matlab to produce all the code to; measure the soil moisture level, create a live reading of the moisture sensor in volts vs time and turn on and off the water pump based on the soil moisture.
RESULTS: My objective was successful; water is pumped when dry soil is detected. When the soil reads wet, the pump stops, and the live moisture sensor in volts vs time seems accurate; I get a reading of when the ground needs water, doesn’t need water, or needs a little more water. 
CONCLUSIONS AND RECOMMENDATIONS:
The Function of the auto watering system was a success. Based on the moisture levels of the soil the pump will either go off or on. I was also able to be updated based on the soil levels if it needed water or not. How I would alter this project if I were to redo it would be to change where the water is pumped to evenly distribute the water throughout the soil.
